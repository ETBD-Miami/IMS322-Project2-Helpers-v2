Image Maps

There is an official "map" element in the HTML spec that can be used to define clickable areas in an image.
It's fairly straight-forward to set up, but has a few disadvantages — namely that the map is not responsive, and it can be a little finicky trying to determine the exact coordinates of the map areas. You can read more about plain HTML image maps here.



Below are a few more interesting and flexible options.

First, if you just need simple shapes like circles or squares, you can just create divs with relative positions that sit over the top of the image. The image and divs should all be inside of the same parent div. Then, you can use an event like "onclick" to call a function. Notice the specific styling that makes this work — I found this example on a random stackoverflow post.

This is one case where I would say that inline styling is approrpiate, especially when it comes to tweaking the percentages. However, it can get very messy. I would suggest trying to move as much of this into your stylesheet as possible (especially if you have a uniform look for all of your hover/click locations).
